# Support

Use GitHub Issues for reproducible bugs and feature requests. Include the
platform-go version, runtime mode, operating system, relevant configuration
(with secrets redacted), logs and the smallest reproduction you can provide.

Use discussions for design questions and integration guidance when that
feature is enabled on the public repository. Do not post credentials, personal
data, private URLs or production stack traces containing sensitive values.

Commercial support, service-level commitments and production operations are not
provided by this repository by default.
