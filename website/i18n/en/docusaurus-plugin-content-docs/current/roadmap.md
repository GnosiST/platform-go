# Roadmap and support boundary

The v0.1 foundation includes capability manifests, sessions, RBAC, organization and role governance, menu trees, resource schemas, audit, file storage, sensitive-data protection, lifecycle and OpenAPI generation.

The default release supports one datasource and one native transaction boundary. Tenant datasource routing, read/write splitting, sharding, federation, XA, MQ and search projections remain independent future nodes and are not certified by capability discovery.

Next priorities are public service contracts and Query/Command Objects, placement and routing safety gates, database certification and disabled-by-default integration ports, then outbox/MQ and asynchronous search projection.
