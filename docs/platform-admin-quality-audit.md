## 跨功能逻辑检查

1. Shell 布局、主题、工作页签和导航必须共享同一套路由/布局状态，不能因切换布局导致菜单重复、页签丢失或权限过滤变化。
2. 通用列表、表单、关系选择器和 Refine provider 必须共同使用资源 Schema 作为类型和权限来源；UI 不能展示后端不存在的操作。
3. 能力配置操作必须经过 Manifest 校验、授权、持久化、审计、缓存失效和刷新回读；本地 React 状态不能作为完成证据。
4. 主题、弹层、Modal、Drawer 和水印层必须在明暗主题下保持对比度、焦点、指针事件和 z-index 正确。
5. 搜索、通知和关系字段必须使用结构化服务端合同、租户范围和取消/错误语义，不能只依赖客户端过滤。
6. 组织、角色、菜单和权限变更必须保留现有 target-mode 迁移闸门；作者工作台完成不代表目标菜单服务和迁移写入已启用。
